# План AGP v1.1

v1.1 начинается после стабилизации v1.0 в production. Фокус версии - lifecycle
пользователя, MFA и уведомления.

## Scope

1. MFA:
   - TOTP для локальных пользователей;
   - recovery codes;
   - политика обязательного MFA по группам;
   - break-glass исключения для аварийных администраторов.

2. Invite/password setup:
   - администратор создает пользователя без финального пароля;
   - AGP отправляет одноразовую ссылку на email;
   - пользователь открывает ссылку, проверяет профиль и задает пароль;
   - invite tokens хранятся как hash и имеют срок действия.

3. Защита чувствительных данных:
   - encrypted storage для sensitive profile fields;
   - key rotation plan;
   - аудит доступа к sensitive fields.

4. Уведомления:
   - настраиваемые notification channels;
   - уведомления о login;
   - уведомления о входе в защищенный ресурс;
   - activity timeline в админке;
   - подготовка audit export для SIEM.

## Открытые Решения

- SMTP provider и TLS policy.
- Какие поля пользователя считаются sensitive.
- Кому отправлять уведомления: пользователю, администраторам или обоим.
- Retention для activity timeline.
- Recovery process при потере MFA device.
- Нужен ли LDAP/AD в v1.1 или отдельной v1.2.
