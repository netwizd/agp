# AGP v1.1 Plan

v1.1 focuses on identity lifecycle and notification workflows after v1.0 is
stable in production.

## Planned Scope

1. MFA:
   - TOTP for local users;
   - recovery codes;
   - admin-enforced MFA policy by group;
   - break-glass admin exemption policy.

2. User invite and password setup:
   - admin creates a user without setting the final password;
   - AGP sends a single-use invite link by email;
   - user opens the link, confirms profile data and sets a password;
   - invite tokens are hashed at rest and expire automatically.

3. Sensitive data protection:
   - encrypted storage for sensitive user profile fields;
   - key rotation plan;
   - strict audit on sensitive field access.

4. Notifications:
   - configurable notification channels;
   - login notifications;
   - protected resource access notifications;
   - admin-visible user activity timeline;
   - audit export path for SIEM later.

## Open Decisions

- SMTP provider and TLS policy.
- Which fields are considered sensitive and must be encrypted.
- Whether notifications go only to admins, to users, or both.
- Retention period for user activity timeline.
- MFA recovery process for lost devices.
